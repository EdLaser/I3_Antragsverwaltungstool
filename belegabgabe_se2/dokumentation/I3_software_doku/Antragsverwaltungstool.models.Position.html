class Position(models.Model):

View In Hierarchy

Represents an application for a special position with variables containing all application data. This application is very similar to the advisory member but with three extra questions. All variables are of type model.field with max lengths and the possibility to be null in the database.

Arguments: models.Model: Django uses models to represent classes to work with in the application, which are representing database tables.
