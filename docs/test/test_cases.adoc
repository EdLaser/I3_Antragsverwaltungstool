== Allgemeine Informationen
=== allgemeine Tests:
* Abfangen ungültiger Eingaben (z.B. Datum)
* korrekte Übertragung der Daten nach Erstellung/Änderung eines Antrages
* Zugriffsberechtigungen für verschiedene Funktionen der Software
* generelle Funktionalität aller beabsichtigten Funktionen
* korrekte Handhabung unerwarteter Fehler
* korrekte Erstellung des TOP, nachdem neuer Antrag gestellt wurde

== Test Cases



*TC01 - Valide Logindaten akzeptiert*
|===
Beschreibung:  Prüfen der Reaktion der Software, wenn valide Logindaten eingegeben werden. Diese müssen akzeptiert werden. Der Login muss gelungen sein.

Vorbedingungen: Das Internet muss verfügbar sein.

Nachbedingungen: Uhrzeit und Logindaten müssen (in einer Datenbank) gespeichert werden.

Erforderliche Daten: Logindaten (Name und Passwort) 
|===

*TC02 - Falsche Logindaten akzeptiert*
|===
Beschreibung: Prüfen der Reaktion der Software, wenn falsche Logindaten eingegeben werden. Entweder ist der Name falsch oder das Passwort ist falsch oder beides ist falsch. Der Login muss fehlerhaft sein.

Vorbedingungen: Das Internet muss verfügbar sein.

Nachbedingungen: Keine

Erforderliche Daten: Logindaten (Name und Passwort)
|===

*TC03 - Leerer Name akzeptiert:*
|===
Beschreibung: Prüfen der Reaktion der Software, wenn der Name leer ist und der Login-Button gedrückt wird. Der Login muss fehlerhaft sein.

Vorbedingungen: Das Internet muss verfügbar sein. 

Nachbedingungen: Keine  

Erforderliche Daten: Keine 
|===

*TC04 - Vollständig ausgefüllter Antrag eingerreicht*
|===
Beschreibung: Prüfen, ob sich ein vollständig ausgefüllter Antrag erreichen lässt. Es muss möglich sein.

Vorbedingungen: Das Internet muss zur Verfüfung stehen. Der Login muss gelungen sein. Die Eingaben (z.B. Datum) müssen gültig sein.

Nachbedingungen: Uhrzeit, Name des Antragsstellers und der Antrag müssen (in einer Datenbank) gespeichert werden.
Erforderliche Daten: vollständig ausgefüllter Antrag
|===

*TC05 - Unvollständiger Antrag eingerreicht*
|===
Beschreibung: Prüfen, ob sich ein unvollständiger Antrag erreichen lässt. Es darf nicht gelungen sein.

Vorbedingungen: Das Internet muss zu Verfügung stehen. Der Login muss gelungen sein. 

Nachbedingungen: Eine Fehlermeldung muss auf dem Bildschirm gezeigt werden. 
Erforderliche Daten: unvollständiger Antrag 
|===

*TC06 - Angezeigter Antrag aufrufbar*
|===
Beschreibung: Prüfen, ob sich ein angezeigter Antrag aufrufen lässt. Es muss möglich sein.

Vorbedingungen: Das Internet muss verfügbar sein. Der Login muss gelungen sein. Die Antragsliste muss auf dem Bildschirm stehen.

Nachbedingungen: Der Inhalt des Auftrag muss dann auf dem Bildschirm gezeigt werden.

Erforderliche Daten: Keine
|===

*TC07 - Externer Nutzer darf keinen Antrag bearbeiten*
|===
Beschreibung: Prüfen, ob ein externer Nutzer einen Antrag bearbeiten kann. Das darf nicht möglich sein.

Vorbedingungen: Das Internet muss verfügbar sein. Der Login muss gelungen sein.

Nachbedingungen: Eine Fehlermeldung muss auf dem Bildschirm gezeigt werden.

Erforderliche Daten: Keine (?)
|===
 
*TC08 - Administrator/Präsidium darf Anträge bearbeiten*
|===
Beschreibung: Prüfen, ob ein interner Nutzer Anträge bearbeiten kann. Das muss möglich sein.

Vorbedingungen: Das Internet muss verfügbar sein. Der Login muss gelungen sein.

Nachbedingungen: Uhrzeit, Benutzername und der bearbeitete Antrag müssen (in einer Datenbank) gespeichert werden.

Erforderliche Daten: Inhalt des Antrags
|===
