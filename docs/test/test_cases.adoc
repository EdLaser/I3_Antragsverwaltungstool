*TC01 - Valide Logindaten akzeptiert*
|===
Beschreibung:  Prüfen der Reaktion der Software, wenn valide Logindaten eingegeben werden. Diese müssen akzeptiert werden. Der Login muss gelungen sein.

Vorbedingungen: Das Internet muss verfügbar sein.

Nachbedingungen: Uhrzeit und Logindaten müssen (in einer Datenbank) gespeichert werden.

Erforderliche Daten: Logindaten (Rolle und Passwort) 
|===

*TC02 - Falsches Passwort akzeptiert*
|===
Beschreibung: Prüfen der Reaktion der Software, wenn falsches Passwort eingegeben wird. Der Login muss fehlerhaft sein.

Vorbedingungen: Das Internet muss verfügbar sein.

Nachbedingungen: Keine

Erforderliche Daten: Logindaten (Rolle und Passwort)
|===

*TC03 - Vollständig ausgefüllter Antrag eingerreicht*
|===
Beschreibung: Prüfen, ob sich ein vollständig ausgefüllter Antrag erreichen lässt. Es muss möglich sein.

Vorbedingungen: Das Internet muss zur Verfüfung stehen. Der Login muss gelungen sein. Die Eingaben (z.B. Datum) müssen gültig sein.

Nachbedingungen: Uhrzeit, Name des Antragsstellers und der Antrag müssen (in einer Datenbank) gespeichert werden.
Erforderliche Daten: vollständig ausgefüllter Antrag
|===

*TC04 - Unvollständiger Antrag eingerreicht*
|===
Beschreibung: Prüfen, ob sich ein unvollständiger Antrag erreichen lässt. Es darf nicht gelungen sein.

Vorbedingungen: Das Internet muss zu Verfügung stehen. Der Login muss gelungen sein. 

Nachbedingungen: Eine Fehlermeldung muss auf dem Bildschirm gezeigt werden. 
Erforderliche Daten: unvollständiger Antrag 
|===

*TC05 - Angezeigter Antrag aufrufbar*
|===
Beschreibung: Prüfen, ob sich ein angezeigter Antrag aufrufen lässt. Es muss möglich sein.

Vorbedingungen: Das Internet muss verfügbar sein. Der Login muss gelungen sein. Die Antragsliste muss auf dem Bildschirm stehen.

Nachbedingungen: Der Inhalt des Auftrag muss dann auf dem Bildschirm gezeigt werden.

Erforderliche Daten: Keine
|===

Erforderliche Daten: Keine
